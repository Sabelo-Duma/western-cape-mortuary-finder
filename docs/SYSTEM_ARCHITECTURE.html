1. The Problem We're Solving

No Centralized Directory
Families in distress have no single place to find mortuaries with available space
Paper-Based Intake
30+ minutes of manual paperwork during the worst moment of their lives
Digital Divide
Many South Africans rely on basic phones without internet access

Our Solution

A full-stack web platform enabling bereaved families across 10 Western Cape cities to locate, compare, and engage mortuary services — with SMS/USSD access, digital intake forms, and multilingual support (English + Afrikaans + isiXhosa).

Key Business Goals

  • Real-time availability — first app in SA showing which mortuaries have space RIGHT NOW
  • Digitize intake — 6-step wizard replaces paper forms (pre-fills DHA-1663 Section A)
  • Bridge the digital divide — SMS endpoint for feature-phone users
  • Revenue via SaaS — 3 subscription tiers for mortuary operators
  • Owner analytics — view counts, contact clicks, intake submissions

6. Security Architecture

Enterprise-grade, 5-layer defense-in-depth model

Layer 1: NETWORK (Vercel Edge) — HTTPS (TLS 1.3), DDoS protection, edge caching
Layer 2: APPLICATION (Next.js) — Zod validation, CSRF (SameSite), XSS (React auto-escaping), service role server-only
Layer 3: AUTHENTICATION (Supabase Auth) — JWT sessions, secure cookie storage, bcrypt password hashing
Layer 4: AUTHORIZATION (PostgreSQL RLS) — Row Level Security on all 6 tables, auth.uid() checks, deny-by-default
Layer 5: DATA (PostgreSQL) — CHECK constraints, NOT NULL, UNIQUE on slugs, FK cascading deletes
